## About

I’m an **Information Systems** student who ships **web systems end to end**: data modeling, APIs, auth and business rules, and responsive front ends. I care about **clarity, structure, and maintainability**—code that teammates can read, extend, and reason about.

**How I work:** start from real workflows, model the data honestly, build thin, testable API boundaries, then iterate with feedback, debugging, and performance checks—not feature churn for its own sake.
