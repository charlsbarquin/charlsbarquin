## About

I’m an **Information Systems** student who builds **web systems end to end**: data modeling, APIs, authentication and domain rules, and responsive interfaces. I aim for software that is **easy to reason about**—predictable state, explicit boundaries, and schemas that match how the business actually works.

**What drives me:** turning messy real-world requirements into **working software**, then tightening it with testing, debugging, and performance checks. I prefer small, verifiable steps over speculative complexity.

> **In one line:** practical systems, honest data models, and UIs that support real workflows—not demo-only screens.

---

## Principles

- **Clarity over cleverness** — readable code paths, obvious naming, and boundaries that match the domain  
- **Data first** — get the model and constraints right; the UI gets easier when the backend tells the truth  
- **Iterate with evidence** — measure, reproduce, fix; avoid rewriting without a reason  
- **Collaborate openly** — share trade-offs early, keep commits and PRs review-friendly
